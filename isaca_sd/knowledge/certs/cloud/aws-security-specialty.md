# AWS Certified Security – Specialty

## Overview

The AWS Certified Security – Specialty (SCS-C02) is Amazon Web Services' advanced certification for practitioners who design, implement, and manage security solutions on the AWS platform. It is a specialty-level credential — sitting above the associate tier — and is intended for security professionals with meaningful AWS hands-on experience. The exam validates cloud-native security skills including identity and access management, threat detection, data protection, incident response, and compliance in AWS environments.

The exam consists of 65 questions completed within 170 minutes. The exam fee is $300 USD. The passing score is approximately 750 out of 1000 points (scaled scoring; the exact threshold can vary slightly by exam form).

## Who Should Get This

This certification is appropriate for security engineers, cloud architects, and security operations professionals who work primarily in AWS environments. It is particularly valuable for practitioners whose organizations run significant workloads in AWS and need a designated security expert who can navigate IAM policies, AWS-native security services (GuardDuty, Security Hub, Macie, CloudTrail, WAF, etc.), and compliance requirements specific to cloud infrastructure.

## Requirements

AWS recommends a minimum of five years of IT security experience in broad domains, plus at least two years of hands-on AWS experience prior to attempting the specialty exam. While not formally enforced, these recommendations are meaningful — candidates without substantial AWS hands-on experience consistently find the exam challenging. Passing the AWS Certified Solutions Architect – Associate or AWS Certified SysOps Administrator – Associate exam first is a common and sensible stepping stone.

AWS certifications renew every three years. Recertification is done by passing the current version of the same exam or a higher-level exam.

## Exam Domains

The SCS-C02 exam covers six domains:

1. Threat Detection and Incident Response (14%)
2. Security Logging and Monitoring (18%)
3. Infrastructure Security (20%)
4. Identity and Access Management (16%)
5. Data Protection (18%)
6. Management and Security Governance (14%)

## Study Tips

Hands-on experience with AWS services is essential and cannot be replicated by reading alone. Build or review real implementations of key services: AWS IAM (roles, policies, SCPs in Organizations), Amazon GuardDuty, AWS Security Hub, AWS Config, AWS CloudTrail, Amazon Macie, AWS WAF and Shield, and AWS KMS. Understanding how these services integrate and how findings flow between them is the foundation of the exam's harder questions.

Official AWS resources include the exam guide, AWS Skill Builder practice exams, and the relevant AWS documentation. Stephane Maarek's course on Udemy is widely recommended by the community. The Adrian Cantrill course is another well-regarded option. Plan for 2–4 months of preparation with a mix of study and hands-on lab work. Create and destroy real AWS environments — use the free tier aggressively and understand exactly what each service logs and where.

## Career Impact

For practitioners working in AWS-heavy organizations, the Security Specialty certification signals advanced cloud security capability to employers. As organizations accelerate cloud adoption, the ability to implement secure AWS architectures, manage cloud identity, and respond to cloud-native incidents has become a dedicated specialty rather than an extension of traditional security roles.

The certification pairs naturally with CISSP for practitioners who want to demonstrate both broad security management competency (CISSP) and deep cloud platform expertise (AWS Security Specialty). In cloud-first organizations and consulting firms that advise AWS customers, the combination is highly marketable.

## How It Compares

AWS Security Specialty vs. AZ-500 (Azure Security Engineer): Both are cloud platform-specific security credentials at roughly the mid-to-advanced level. The right choice depends entirely on your organization's cloud platform. AWS and Azure have meaningful market share differences by industry vertical — AWS leads in startups and technology companies; Azure dominates in enterprise, government, and defense contractor environments. Practitioners working in multi-cloud environments or moving between roles benefit from holding both.

AWS Security Specialty vs. CCSP: CCSP (Certified Cloud Security Professional, ISC2) is a vendor-neutral cloud security credential targeting architecture and governance. AWS Security Specialty is platform-specific and more technically operational. Both have market value; CCSP is the choice for practitioners who advise across cloud providers; AWS Specialty is the choice for depth within the AWS platform.
